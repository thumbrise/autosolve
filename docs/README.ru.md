# GitHub Dispatcher Agent – автоматический AI-ассистент для ваших репозиториев

## Идея

Мы создаем простой и надёжный инструмент, который самостоятельно отслеживает активность в GitHub-репозитории и при необходимости запускает AI-агентов для решения задач, таких как исправление багов, добавление новых функций или анализ кода. Продукт представляет собой автономный процесс, который работает по принципу периодического опроса (polling), не требует сложной настройки вебхуков или CI/CD и может быть развёрнут где угодно – от личного ноутбука до выделенного сервера.

## Проблема

Современные AI-инструменты для автоматизации разработки (OpenHands, SWE-agent, Devin) обладают мощными возможностями, но их интеграция с GitHub часто оказывается сложной или нестабильной:

- требуется настройка вебхуков или GitHub Actions, что не всегда удобно.
- встроенные планировщики (cron) работают ненадёжно или требуют дополнительной инфраструктуры.
- большинство решений ориентированы на однократный запуск по команде, а не на постоянное фоновое наблюдение.
- для небольших команд или индивидуальных разработчиков развёртывание таких агентов может быть слишком сложным и дорогим.

## Наше решение

Мы предлагаем легковесного диспетчера, который:

- запускается как постоянный процесс (демон) на вашем компьютере или сервере.
- периодически (например, раз в 5–10 минут) опрашивает GitHub API для заданного репозитория.
- обнаруживает новые issues, pull requests, комментарии, упоминания и метки.
- сохраняет состояние последней проверки в локальном хранилище (выбор конкретной технологии остаётся открытым – это может быть файл, встроенная БД или что-то ещё).
- при наступлении событий, подходящих под заданные правила (например, issue с меткой `ai` или комментарий с `@bot`), вызывает внешний AI-инструмент (например, ra-aid) и передаёт ему контекст задачи.
- после завершения работы AI публикует результат обратно в GitHub – создаёт комментарий, PR или выполняет другое действие.

Все решения о конкретных технологиях (язык программирования, тип хранилища, способ деплоя) мы сможем принять позже, когда лучше поймём требования. Архитектура спроектирована так, чтобы заменять компоненты без переписывания всей системы.

## Как это поможет пользователям

- **Простота**: не нужно разбираться в вебхуках или GitHub Actions – просто запустил и забыл.
- **Надёжность**: поллинг с сохранением состояния гарантирует, что ни одно событие не будет пропущено даже после перезапуска.
- **Гибкость**: можно использовать любые AI-инструменты, которые умеют работать из командной строки (ra-aid, локальные модели через Ollama, скрипты на Python и т.д.).
- **Экономия ресурсов**: агент потребляет минимум памяти и процессора, может работать даже на Raspberry Pi.
- **Безопасность**: все данные остаются у вас, токены не передаются третьим лицам.

## Высокоуровневая архитектура

Система состоит из нескольких логических блоков, которые могут быть реализованы разными способами:

1. **Диспетчер** – главный цикл, который запускается по расписанию. Он отвечает за координацию остальных модулей.
2. **Сборщик данных** – модуль, который через GitHub API получает текущее состояние репозитория (issues, PR, комментарии).
3. **Хранилище состояний** – компонент, который сохраняет информацию о том, какие события уже были обработаны, чтобы избежать дублирования.
4. **Анализатор правил** – проверяет, подходит ли новое событие под заданные критерии (лейблы, авторы, ключевые слова).
5. **Исполнитель** – запускает внешний AI-инструмент с нужными аргументами и обрабатывает его вывод.
6. **Публикатор** – отправляет результат обратно в GitHub (комментарий, создание PR и т.п.).

Каждый из этих блоков может быть реализован независимо, и мы сможем заменять или улучшать их по мере развития проекта.

## Пример сценария работы

1. Пользователь настраивает агент на свой репозиторий, указывая токен и интервал проверки.
2. Агент начинает работу. При каждом цикле он запрашивает все открытые issues и pull requests.
3. Если появляется новый issue с меткой `ai`, агент замечает это и передаёт его описание в ra-aid (или другой инструмент).
4. ra-aid анализирует код, генерирует исправление и создаёт PR.
5. Агент получает результат и публикует в issue комментарий со ссылкой на созданный PR.
6. Пользователь видит, что задача решена автоматически.

## Текущий статус и ближайшие шаги

- Мы находимся на этапе формирования концепции и поиска подходящей реализации.
- Планируется создать прототип на одном из популярных языков (Go, Python) с минимальным функционалом: поллинг, сохранение состояния, запуск внешней команды.
- После проверки работоспособности добавим поддержку правил и публикацию результатов.
- В дальнейшем возможно создание готовых сборок (Docker-образ, бинарные релизы) и документации для самостоятельного развертывания.

## Почему этот проект имеет смысл

Автоматизация рутинных задач в разработке – актуальная потребность многих команд. Готовые AI-агенты уже существуют, но их интеграция с экосистемой GitHub часто оставляет желать лучшего. Наш подход заполняет этот пробел, предлагая простое и надёжное связующее звено между GitHub и любыми AI-инструментами. Это позволяет разработчикам сосредоточиться на более важных вещах, пока агент занимается решением типовых задач.

Мы не изобретаем AI заново – мы делаем его доступным и удобным в использовании.
